CodeBrix.VideoProcessing.OpenCV5 is a .NET binding for OpenCV 5 — image
processing, video capture and analysis, camera calibration, object detection,
machine learning, and the OpenCV contrib extra modules. It is a fork/port of
the OpenCvSharp project (its OpenCvSharp5 package family) and additionally
redistributes the exact native binary libraries that the upstream project
published to NuGet. Everything incorporated from other projects is listed
below.

Jeremy Ellis and contributors license the CodeBrix.VideoProcessing.OpenCV5
project to you under the terms set forth in the LICENSE file, which is located
at the root of this repository.

If you have any questions about any third-party code or services referenced
here, or if you believe that any third-party code has been included in
CodeBrix.VideoProcessing.OpenCV5 in error, please open a new issue at:
https://github.com/ellisnet/CodeBrix.VideoProcessing.OpenCV5/issues

   Non-trivial modifications made during the port:
     - Namespaces renamed from OpenCvSharp[.*] to CodeBrix.VideoProcessing.OpenCV5[.*]
       (OpenCvSharp.WpfExtensions -> CodeBrix.VideoProcessing.OpenCV5.Wpf).
       Type names were NOT renamed (OpenCvSharpException, OpenCvSafeHandle,
       and the native library name OpenCvSharpExtern are retained).
     - Retargeted from net8.0 to net10.0; <ImplicitUsings> removed and the
       previously-implicit using directives added explicitly to each file.
     - Strong-naming removed (Key.snk not ported); Properties/AssemblyInfo.cs
       and OpenCvSharp.dll.config (a Mono dllmap) not ported.
     - Dead code stripped: Modules/features/DenseFeatureDetector.cs (whole file
       was wrapped in `#if false` upstream) and the empty file
       Modules/ximgproc/SelectiveSearch/SelectiveSearchSegmentationStrategy.cs
       were not ported.
     - Modules/stitching/Stitcher.cs: the nested `namespace Detail` block was
       split out into Modules/stitching/StitcherDetail.cs so both files use
       file-scoped namespaces.
     - The WPF converters' `#if WINDOWS && (NET48 || NETCOREAPP3_1_OR_GREATER)`
       guards were simplified to `#if WINDOWS` (the framework clause is always
       true on .NET 10).
     - XML doc comments were added to the four Roslyn analyzer classes' public
       members (upstream suppressed CS1591 project-wide for that project).
     - test/OpenCvSharp.Tests.Windows/BitmapConverterTest.cs was NOT ported (it
       tests the GdipExtensions package, which was eliminated); the WaitAnyLinux
       test was converted to an unconditional documented skip (latent upstream
       bug — see the comment in tests/.../videoio/VideoCaptureTests.cs).
     - The following upstream projects/packages were deliberately NOT ported
       (elimination decided 2026-07-07): OpenCvSharp.GdipExtensions, the
       .slim runtime variants, the wasm runtime, the uwp runtime, the 32-bit
       linux-arm runtime, and the OpenCvSharp5.Windows meta-package.
     - NuGet packaging metadata, versioning scheme, icon, and readme were
       replaced with the CodeBrix family equivalents.

2. OpenCvSharp5 native runtime binaries (OpenCV 5 + OpenCvSharpExtern)

   The files stored xz-compressed under native_libraries/runtimes/ are the
   EXACT binaries published by the upstream OpenCvSharp project in its NuGet
   runtime packages, extracted unmodified from these upstream packages:

     OpenCvSharp5.runtime.win                 5.0.0.20260703
     OpenCvSharp5.runtime.win-arm64           5.0.0.20260703
     OpenCvSharp5.official.runtime.linux-x64  5.0.0.20260703
     OpenCvSharp5.runtime.linux-arm64         5.0.0.20260703
     OpenCvSharp5.runtime.osx.x64             5.0.0.20260704
     OpenCvSharp5.runtime.osx.arm64           5.0.0.20260704

   Per-file SHA-256 hashes of the raw (uncompressed) binaries are recorded in
   native_libraries/SHA256SUMS.txt; the pack step verifies them before any
   package is produced. These binaries were captured once (2026-07-07) and are
   never re-fetched from the upstream project or nuget.org.

   The native libraries are builds of OpenCV and the OpenCV contrib modules
   (https://github.com/opencv/opencv, https://github.com/opencv/opencv_contrib):

   Copyright:  Copyright (C) 2000-2026, Intel Corporation, OpenCV Foundation,
               and the OpenCV contributors
   License:    Apache License 2.0 (OpenCV 4.5+ / 5.x; full text below)

   together with the OpenCvSharpExtern C wrapper layer by shimat (Apache
   License 2.0, see item 1).

   NOTE — additional statically/dynamically linked components: as with the
   official upstream OpenCV and OpenCvSharp binary distributions, these native
   binaries embed OpenCV's bundled third-party components (e.g. libjpeg-turbo,
   libpng, zlib, protobuf) under their respective permissive licenses, and
   include FFmpeg-based video I/O support: on Windows as the separate
   opencv_videoio_ffmpeg500_64.dll plugin, and in the Linux builds embedded
   within libOpenCvSharpExtern.so. FFmpeg is licensed under the LGPL-2.1+
   (see https://ffmpeg.org/legal.html). These are binary redistributions of
   the unmodified upstream-published artifacts; consult the upstream OpenCV
   project's 3rdparty documentation for the complete component inventory.
